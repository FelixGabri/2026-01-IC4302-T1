Este fue mas distino a los demas. Se ocupa crear un service monitor

Las bases mongo,elastic,maria, redis, postfes usan exportes dedicados, no las metricas de flask.

Couch es la expepcion, no tiene, asi que hay que configurarlo uno.

Hay aplicar las metricas de Flask por ServiceMonitor.

1. aplicar un label al servicio porque el port inicial es no tienen nombre:

kubectl label svc flasktest app=flask -n default

2.Se modifico flask.yaml(TC1/charts/app/templates) para agregar el service monitor.

Dos cambios importantes respecto al original:

Se agregó labels: app: {{ .Values.config.flask.name }} al Service — antes no tenía labels, solo selector. El ServiceMonitor necesita matchear por label del Service, no del pod.
Se agregó el ServiceMonitor al final.



Se tuvo que agrear esta parte a app.py. el cliente de prometheus ocupa que la respuesta tenga el content type correcto, en flash hay que hacerlo explicito

from prometheus_client import  CONTENT_TYPE_LATEST

@app.route("/metrics")
def metrics():
   return generate_latest(), 200, {'Content-Type': CONTENT_TYPE_LATEST}
